# Управление проектом и риски

## 1. Реестр рисков

| Риск | Вероятность / Влияние | Обнаружение | Защита | Остаточный риск |
|------|------------------------|--------------|--------|------------------|
| **Зацикливание агента (бесконечные уточнения)** | Низкая / Высокое | Счётчик итераций | Жёсткий лимит (max 3 шага); если превышен — принудительный выход с генерацией диалога | Низкий |
| **Накопление контекста превышает лимит токенов** | Средняя / Среднее | Проверка длины истории перед запросом | Очистка старых шагов (keep last N) или суммаризация | Средний — может потеряться часть информации |
| **Неверное решение агента (ненужный поиск)** | Средняя / Низкое | Анализ логов | Улучшение промпта; можно добавить «стоимость» действия в рассуждения | Низкий |
| **Поиск возвращает нерелевантные результаты** | Высокая / Среднее | Агент сам оценивает релевантность (промпт просит проверить) | Если результаты нерелевантны, агент может переформулировать запрос | Средний — качество зависит от поисковика |
| **Превышение бюджета из-за множественных итераций** | Низкая / Высокое | Мониторинг стоимости каждого шага | Лимит итераций; предупреждение при достижении порога | Низкий |
| **Сбой API OpenAI / ElevenLabs** | Средняя / Высокое | Проверка HTTP‑статуса, таймауты | Повторные попытки (3 раза с экспоненциальной задержкой), логирование ошибки | Низкий — при временных сбоях видео не создастся, но процесс можно перезапустить |
| **Некорректный диалог от LLM** | Средняя / Среднее | Проверка структуры JSON, наличия реплик | Валидация схемы ответа; если структура нарушена — повтор запроса с уточнением | Низкий — промпт составлен так, чтобы гарантировать JSON |
| **Повреждение временных файлов** | Низкая / Среднее | Проверка размера созданных файлов | Удаление временной папки после завершения; если файл пустой — перегенерация | Низкий |
| **Низкое качество синтеза речи (роботизированный голос)** | Средняя / Низкое | Субъективная оценка создателя | Настройка параметров stability/similarity в ElevenLabs; использование лучших голосов | Низкий — можно заменить голос вручную |
| **Отсутствие фоновых видео в папке** | Низкая / Высокое | Проверка наличия файлов перед запуском | Добавить fallback (одно базовое видео в комплекте) | Низкий |

## 2. Политика логирования и работа с данными

- **Логи**: все действия записываются в файл `logs/pipeline.log` с ротацией (размер 10 МБ, 5 backups). Формат: `timestamp - level - module - message`.
- **Персональные данные**: проект не собирает и не хранит никакие персональные данные пользователей. Темы для видео задаются администратором и не содержат личной информации.
- **API‑ключи**: хранятся в переменных окружения (файл `.env`) и не попадают в логи.
- **Сгенерированные видео**: сохраняются локально, доступ к ним ограничен правами файловой системы.

## 3. Защита от инъекций и подтверждение действий

- **Prompt injection**: может произойти через результаты поиска, если они содержат вредоносные инструкции. Защита: результаты поиска передаются в LLM как простой текст в поле «observation», и системный промпт чётко отделяет роль агента.
- **Подтверждение действий**: агент действует автономно, но все его действия логируются. В будущем можно добавить режим «с подтверждением» перед реальным поиском.
- **Безопасность временных файлов**: все временные файлы создаются в уникальной подпапке `temp/<run_id>` с правами 700, после завершения папка удаляется.
- **Доступ к API**: ключи имеют минимально необходимые права (только генерация текста и речи).
