# Продуктовое предложение EduVid Factory

## 1. Обоснование идеи
Создание качественного короткого образовательного видео сейчас требует ручной работы сценариста, диктора, дизайнера и монтажёра. Это долго и дорого. В то же время популярность формата «диалог двух персонажей на фоне игры» (Minecraft / Subway Surfers) огромна — такие ролики набирают миллионы просмотров. Автоматизация этого процесса позволит создавать контент в промышленных масштабах с минимальными затратами.

## 2. Цель и метрики

**Цель проекта** — разработать систему, которая за 5 минут и дешевле $2 создаёт минутное видео с диалогом двух персонажей на заданную тему.

**Продуктовые метрики успеха:**
- Время создания одного видео (от темы до готового файла) — не более 5 минут.
- Себестоимость одного видео — менее $2.
- Процент успешных генераций (без ошибок API) — > 80%.

**Технические метрики:**
- p95 latency генерации аудио (одна реплика) — < 3 секунд.
- p95 latency рендера видео — < 3 минут для 1‑минутного ролика.
- Доступность API (OpenAI, ElevenLabs) — в рамках их SLA.

## 3. Сценарии использования

### Основной сценарий
Пользователь (администратор) указывает тему, например «градиентный бустинг». Скрипт генерирует диалог, создаёт аудио, выбирает фон, монтирует видео и сохраняет его. Готовый ролик можно вручную загрузить в соцсети.

### Edge‑кейсы
- **Ошибка одного из API** — скрипт повторяет запрос до 3 раз, потом логирует ошибку и останавливается.
- **Тема слишком сложная или непонятная модели** — диалог может получиться бессвязным. Выход: предусмотреть ручной контроль или доработку промпта.
- **Длина диалога превышает лимиты** — если диалог получается дольше 90 секунд, можно обрезать или разбить на несколько видео.
- **Нет подходящего фона** — скрипт использует последний доступный фон или выдаёт ошибку.

## 4. Ограничения

**Технические:**
- Видео генерируется на локальной машине; для работы требуется установленный Python 3.10+, FFmpeg и библиотеки (moviepy, requests и др.).
- Зависимость от сторонних API: OpenAI и ElevenLabs (необходимы ключи и оплата).
- При большом количестве запросов возможны ограничения скорости (rate limits) со стороны API.

**Операционные:**
- Бюджет на API — от $0.4 до $2 за видео (зависит от длины и тарифов).
- Для работы нужен компьютер с достаточной производительностью для рендера видео (рекомендуется 8+ ГБ RAM).
- Голоса в ElevenLabs должны быть заранее созданы (вымышленные персонажи).

## 5. Архитектурный набросок

Система состоит из последовательных модулей:
[Тема] → [Генератор сценария (OpenAI)] → [Парсер реплик] → [TTS (ElevenLabs)] → [Сборка аудио] → [Выбор фона/фото] → [Видеокомпозитор (MoviePy)] → [Готовое видео]


**Модули:**
1. **script_generator.py** — запрос к OpenAI, получение JSON с диалогом.
2. **audio_generator.py** — для каждой реплики вызов ElevenLabs, сохранение временных MP3, объединение.
3. **asset_selector.py** — случайный выбор фонового видео и фото персонажей из локальных папок.
4. **video_composer.py** — наложение слоёв (фон, два фото, аудио) через MoviePy.
5. **main.py** — оркестратор, логирование, обработка ошибок.

**Интеграции:**
- OpenAI API (ChatCompletion)
- ElevenLabs API (text-to-speech)
- Локальная файловая система

## 6. Data Flow и роль LLM

- **LLM (OpenAI)** отвечает за творческую часть: написание диалога в заданном стиле. Это единственное место, где используется большая языковая модель.
- **ElevenLabs** — преобразование текста в речь, никакой генерации текста.
- **Выбор фона и фото** — детерминированный или случайный (без LLM).
- **Сборка видео** — чисто алгоритмическая задача (MoviePy).

Таким образом, LLM используется только для генерации сценария; остальные шаги жёстко запрограммированы.
