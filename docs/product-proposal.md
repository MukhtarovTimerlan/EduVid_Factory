# Продуктовое предложение EduVid Factory (агент с ReAct-циклом)

## 1. Обоснование идеи
Автоматическая генерация видео часто страдает от поверхностности: LLM выдаёт общие фразы, не углубляясь в тему. Чтобы контент был действительно полезным и точным, системе нужно уметь:
- искать актуальную информацию,
- проверять её полноту,
- при необходимости уточнять.

Агентный подход с циклом «рассуждение — действие — наблюдение» (ReAct) позволяет имитировать работу человека-сценариста: сначала собрать данные, потом осмыслить, потом написать текст, и если чего-то не хватает — вернуться к поиску.

## 2. Цель и метрики

**Цель проекта** — разработать агента, который за 8–10 минут и менее $2 создаёт минутное обучающее видео, используя многократный поиск в интернете и саморефлексию для повышения качества диалога.

**Продуктовые метрики успеха:**
- Время создания одного видео — не более 10 минут (с учётом итераций).
- Себестоимость одного видео — менее $2.
- Субъективная оценка качества диалога (по шкале 1–5) — не ниже 4 (оценивается экспертом на выборке).
- Количество итераций поиска в среднем — 1.5 (не более 3).

**Технические метрики:**
- p95 latency поискового запроса — < 2 сек.
- p95 latency LLM (на один шаг рассуждения) — < 5 сек.
- p95 latency полного цикла агента — < 8 мин.
- Надёжность (успешное завершение без ошибок) — > 85%.

## 3. Сценарии использования

### Основной сценарий
Пользователь задаёт тему. Агент:
- Формирует первоначальный запрос к поисковой системе.
- Получает результаты.
- Анализирует: достаточно ли информации для написания диалога? Если нет — уточняет запрос и ищет снова (например, добавляет «простое объяснение» или «примеры»).
- После сбора достаточных данных генерирует диалог.
- Передаёт диалог в конвейер синтеза и монтажа.

### Edge‑кейсы
- **Поиск возвращает противоречивую информацию** — агент запрашивает дополнительные источники или выбирает наиболее частотное определение.
- **Тема слишком новая, в поиске мало данных** — агент может использовать общие знания LLM как fallback, но отмечает это в логе.
- **Агент зацикливается (всё время уточняет)** — предусмотрен лимит итераций (3), после чего генерируется диалог на основе имеющегося.
- **Пользователь хочет уточнить стиль (например, «для детей»)** — можно передать дополнительный параметр, агент учтёт его в рассуждениях.

## 4. Ограничения

**Технические:**
- Агент работает последовательно, нет параллельных запросов.
- Зависимость от трёх API (поиск, OpenAI, ElevenLabs).
- Для работы агента нужен достаточный контекст (история запросов помещается в токены).

**Операционные:**
- Бюджет на API может вырасти из-за дополнительных итераций (до $2 за видео).
- Требуется более сложная логика обработки ошибок и контроля циклов.

## 5. Архитектурный набросок (ReAct-агент)

[Тема] → [Агент (ReAct цикл)]  
│  
├─ [Планирование] → [Поиск (инструмент)] → [Наблюдение]  
│ ↑ │  
│ └─────────────────────────────────────┘  
│ (повтор при необходимости)  
│  
└─ [Генерация финального диалога] → [TTS] → [Видео]  
  
**Компоненты:**

1. **agent_core.py** — основной цикл ReAct:
   - Системный промпт, описывающий роль «сценариста-исследователя».
   - Хранит историю сообщений (мысли, действия, наблюдения).
   - На каждом шаге вызывает LLM с инструкцией: либо сформулировать поисковый запрос, либо завершить и написать диалог.
2. **tools/search.py** — функция, вызывающая поисковый API и возвращающая текстовые сниппеты.
3. **memory.py** — хранит историю запросов и результатов в рамках сессии (простой список).
4. **script_generator.py** — после завершения цикла получает финальный диалог (в формате JSON) и передаёт дальше.
5. **audio_generator.py** — для каждой реплики вызов ElevenLabs, сохранение временных MP3, объединение.
6. **asset_selector.py** — случайный выбор фонового видео и фото персонажей из локальных папок.
7. **video_composer.py** — наложение слоёв (фон, два фото, аудио) через MoviePy.

**Интеграции:**
- OpenAI API (для рассуждений и генерации диалога)
- Поисковый API (Google / Bing / SerpAPI)
- ElevenLabs
- Локальная файловая система

## 6. Data Flow и роль LLM

- **LLM — ядро агента**: на каждом шаге получает историю взаимодействий и решает, что делать дальше.
- **Инструмент (поиск)** — вызывается агентом по мере необходимости.
- **Память** — история всех шагов (мысли, действия, результаты) подаётся в контекст LLM при каждом новом вызове.
- **Выход** — после сигнала от агента «готово» получаем JSON с диалогом.
- **ElevenLabs** — преобразование текста в речь, никакой генерации текста.
- **Выбор фона и фото** — детерминированный или случайный (без LLM).
- **Сборка видео** — чисто алгоритмическая задача (MoviePy).

Таким образом, система становится полноценным агентом, способным к многошаговому рассуждению и адаптации.
